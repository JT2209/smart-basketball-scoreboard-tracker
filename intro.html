Project Title

The Smart Basketball Scoreboard Tracker

Mission

We take care of gaming statistics very seriously, in an easy way.

Project Pitch

Since 2014, I've been playing basketball for WDSRA and Special Olympics and for every game, a scorekeeper runs them where the clock, fouls, and timeouts are all tracked instantly with a clean, professional digital display--with no expensive hardware. The Smart Basketball Scoreboard Tracker is a web-based application that alllows schools, recreation leagues, and community gyms to manage games in real time using exclusively a laptop or tablet. With a simple interface, scorekeepers can update scores, control the game clock, and track key stats while spectators view a live scoreboard on a large screen. The system also gaves game history, making it simple to review past performances and improve team strategy. This project provides a low-cost, accessible alternative to traditional scoreboards while improving accuracy and organization. With future expansion potential such as live online viewing and a resposive design, this tool can grow into a powerful sports management platform.

Users

The main users are scorekeepers and game operators who take control of the scoreboard and input game data. Coaches also interact with the system by viewing accurate scores for each game. For the players, they will benefit from a more organized and professional game experience. Spectators will also view a clear, real-time scoreboard display when the scoreboard tracker is working. Schools and recreation leagues use the system as an affordable scoring solution.

Modality

This project is a web application. When the scoreboard is responsive, it is accessible through a browser on desktop computers or laptops. Later, the system could be expanded into a mobile app and a live online scoreboard for remote viewing.

Technologies

  • Backend: Node.js with Express or Python with Flask
  • Frontend: UI/UX Design, Python, HTML, CSS, JavaScript
  • Database: MySQL or SQLite
  • Development Environment: Visual Studio Code, Postman, GitHub

Data

The application stores internally generated data that is used. This means that the user creates all data during operation. Data in the project will include the information for teams, games, scoring, fouls and timeouts, and player entries. This information will be created manually by the scorekeeper or operator. The system will then automatically generate it during gameplay. The data will be stored in SQL, which will be organized into tables like teams, games, scores, and stats. This system will not handle maximum personal data. However, basic security rules will be applied. They include input validation that will protect bad/incorrect data, restricted database access through backend APIs, and environment variables will protect credentials. The system will take care of small to medium datasets.
